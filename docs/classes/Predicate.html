_afterwards

Function

Defined in src/Predicate.js:133

_afterwards is immediately after the sentence stops. If it returns a sentence or an array of sentences, these will be executed simultaneously

_begin

Function

Defined in src/Predicate.js:101

_begin is called directly after the sentence happens. So far, the return value is ignored.

_expand

Function

Defined in src/Predicate.js:116

_expand works in a similar way to _prepare except it is called immediately after a sentence happens. If it returns a sentence, or an array of sentences, these will be executed consequetively and the main sentence will be stopped after the last one finishes.

_prepare

Function

Defined in src/Predicate.js:77

_prepare is called before a sentence happens. If it returns a sentence or list of sentences, these sentences will be executed consequetively before the original sentence happens.

banal

Boolean

Defined in src/Predicate.js:140

If a predicate is marked banal, sentences using it will be ignored by certain processes to do with story telling.

Default: false

check

Function

Defined in src/Predicate.js:91

check is called to decide whether it is necessary to call _begin. If it returns truthy then _begin will be skipped, the start process will not be cancelled however. Its secondary purpose is for answering question sentences (true/false) when they have not been specifically declared as sentences.

problem

Function

Defined in src/Predicate.js:85

Problem returns truthy if the sentence is illegal.

skipIf

Function

Defined in src/Predicate.js:60

skipIf is called as when starting a sentence. If it returns a truthy value then the sentence will cancel starting and won't happen. Should generally be used to check whether an action is unnecessary because its outcome is already true.

until

Function

Defined in src/Predicate.js:125

until is called immediately after a sentence happens (after _expand). It has an additional callback arguemnt (prepended) which, when called will stop the sentence.
